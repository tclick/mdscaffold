# Support

`python-project` is a personal project maintained by a single developer in spare time. Support is provided on a best-effort basis.

---

## Before Opening an Issue

1. Check the [README](README.md) to make sure the feature or usage you expect is documented
2. Check the [open and closed issues](https://github.com/tclick/python-project/issues?q=is%3Aissue) — your question may already be answered
3. Confirm you are running Python 3.13+ and a recent install of `python-project`

---

## How to Get Help

**Bug reports and questions** — open a [GitHub issue](https://github.com/tclick/python-project/issues). Please include:

- Python version (`python --version`)
- How you installed `python-project` and which version/commit
- The full command you ran and the complete output or error

**Feature requests** — also via GitHub issues, labelled `enhancement`.

---

## What is Out of Scope

- General Amber MD support or troubleshooting (see the [Amber mailing list](https://lists.ambermd.org/mailman/listinfo/amber))
- General Python environment or dependency issues unrelated to `python-project` itself

---

## Response Time

There is no guaranteed response time. Issues are reviewed when time allows, typically within a few days to a couple of weeks.
